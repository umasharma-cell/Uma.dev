# Veterinary Chatbot SDK

[Permalink: Veterinary Chatbot SDK](https://github.com/umasharma-cell/Veterinary-Chatbot#veterinary-chatbot-sdk)

A production-ready, embeddable chatbot SDK for veterinary clinics that provides AI-powered pet care assistance and appointment booking capabilities.

## Features

[Permalink: Features](https://github.com/umasharma-cell/Veterinary-Chatbot#features)

- **AI-Powered Responses**: Uses Google Gemini API for intelligent veterinary Q&A
- **Appointment Booking**: Conversational flow for scheduling vet appointments
- **Easy Integration**: Single script tag embedding for any website
- **Session Management**: Persistent conversations across page refreshes
- **Context Support**: Optional configuration for personalized experiences
- **Responsive Design**: Works seamlessly on desktop and mobile devices
- **Clean Architecture**: Separated concerns with MVC pattern

## Installation

[Permalink: Installation](https://github.com/umasharma-cell/Veterinary-Chatbot#installation)

### 1\. Clone the Repository

[Permalink: 1. Clone the Repository](https://github.com/umasharma-cell/Veterinary-Chatbot#1-clone-the-repository)

```
git clone https://github.com/yourusername/veterinary-chatbot.git
cd veterinary-chatbot
```

### 2\. Backend Setup

[Permalink: 2. Backend Setup](https://github.com/umasharma-cell/Veterinary-Chatbot#2-backend-setup)

```
cd BACKEND
npm install

# Create .env file with your credentials
cp .env.example .env
# Edit .env and add your Gemini API key and MongoDB URI
```

### 3\. Frontend Setup

[Permalink: 3. Frontend Setup](https://github.com/umasharma-cell/Veterinary-Chatbot#3-frontend-setup)

```
cd FRONTEND
npm install
```

### 4\. Start MongoDB

[Permalink: 4. Start MongoDB](https://github.com/umasharma-cell/Veterinary-Chatbot#4-start-mongodb)

```
# If using local MongoDB
mongod

# Or use MongoDB Atlas cloud service
```

### 5\. Run the Application

[Permalink: 5. Run the Application](https://github.com/umasharma-cell/Veterinary-Chatbot#5-run-the-application)

**Backend (Terminal 1):**

```
cd BACKEND
npm run dev
# Server runs on http://localhost:5000
```

**Frontend (Terminal 2):**

```
cd FRONTEND
npm run dev
# UI runs on http://localhost:5173
```

## Key Design Decisions

[Permalink: Key Design Decisions](https://github.com/umasharma-cell/Veterinary-Chatbot#key-design-decisions)

### 1\. Separation of Concerns

[Permalink: 1. Separation of Concerns](https://github.com/umasharma-cell/Veterinary-Chatbot#1-separation-of-concerns)

- **Controllers**: Handle HTTP requests/responses
- **Services**: Business logic and external API integration
- **Models**: Data structure and validation
- **Routes**: Endpoint definitions

### 2\. Rule-Based Appointment Booking

[Permalink: 2. Rule-Based Appointment Booking](https://github.com/umasharma-cell/Veterinary-Chatbot#2-rule-based-appointment-booking)

- Deterministic flow instead of AI-driven to ensure reliability
- State machine approach for predictable user experience
- Clear validation at each step

### 3\. Session Management

[Permalink: 3. Session Management](https://github.com/umasharma-cell/Veterinary-Chatbot#3-session-management)

- UUID-based sessions without user authentication
- LocalStorage for client-side persistence
- MongoDB for server-side conversation storage

### 4\. Veterinary-Only Responses

[Permalink: 4. Veterinary-Only Responses](https://github.com/umasharma-cell/Veterinary-Chatbot#4-veterinary-only-responses)

- Strict system prompt to limit AI responses
- Keyword detection for veterinary topics
- Polite rejection of off-topic questions

## Trade-offs & Assumptions

[Permalink: Trade-offs & Assumptions](https://github.com/umasharma-cell/Veterinary-Chatbot#trade-offs--assumptions)

### Trade-offs Made

[Permalink: Trade-offs Made](https://github.com/umasharma-cell/Veterinary-Chatbot#trade-offs-made)

1. **Iframe vs Direct Embed**: Chose both options - iframe for isolation, direct for performance
2. **Session Storage**: LocalStorage over cookies for simplicity
3. **Validation**: Basic validation to avoid over-engineering
4. **UI Framework**: React for component reusability despite bundle size

### Assumptions

[Permalink: Assumptions](https://github.com/umasharma-cell/Veterinary-Chatbot#assumptions)

1. Users have modern browsers with JavaScript enabled
2. MongoDB is accessible (local or cloud)
3. Gemini API has reasonable latency
4. Single language support (English) is sufficient
5. No authentication required for chat access
